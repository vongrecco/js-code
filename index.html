Zatwierdzony - po git commit

  1. git init inicjalizacja repozytorium
  2. git status status repozytorium
  3. .gitignore tworzenie pliku w którym umieszczamy ignorowane pliki

git add

  1. git add dodaje pliki do kolejki
  2. mogę w nim określić który plik, podając jego nazwę, folder, lub wszystkie pliki za pomocą "."
  3. git status ponownie pokaże jakie zmiany zaszły w repo

git commit

  1. git commit zapisuje zmiany w repozytorium
  2. git commit -m "komentarz zmian"powinien zawierać dodatkowo jakie zmiany zostały wprowadzone
  3. czasem jeśli zamominymy o podaniu komentarza to otwrozy się okienko, w którym będziemy mogli go dodać

Notatki

Bilokacja - plik posiada dwa stany jednocześnie. Występuje jeśli tworzymy plik, dodajemy git add, a następnie przed commitem wprowadzamy w nim zmiany. W takim wypadku w "przechowalni" znajduje się wersja przed ostatniej modyfikacji, a w katalogu roboczym wersja ze zmianąs

Mozna wtedy zatwierdzić poprzednie zmiany poprzez git commit lub dodać najnowsze zmiany za pomocą git add a następnie git commit

Przestrzeń robocza i stage

  1. git clean służy do usuwania niesledzonych plików. Nieśledzone to takie które nie były dodawane do repo za pomocą git add.
  2. po użyciu tej komendy pojawi się błąd w którym dostatniemy informację, że zmiany te są nieodwracalne
  3. git clean -n - pokazuje jakie pliki zostałby usunięte
  4. git clean -nd - z flaga nd pokazuje jakie pliki i (d) foldery może usunąć - domyslnie (n) ze względów bezpieczeństaw dotyczy tylko plików
  5. git clean -idf
  6. i - to tryb interaktywny
  7. d - uwzględnia katalogii
  8. f - faktycznie usówa pliki
  9. git clean -xn usuwa pliki sytemowe
  10. pokazuje się kilka opcji np. usuń, filtruj, zapytaj które itd. wybieramy odpowidni numerek i potwierdzamy y/n
  1. git reset usuwa pliki kolejki oczekiwania
  2. git rm nazwa.html usuwa plik zostaje usunięty, następnie git commit aby usunąć z całkowicie
  3. git checkout ściezka - przywraca plik z repozytorium
